## Process

1. Read ROADMAP.md for the phase's requirements
2. Read CONTEXT.md for locked user decisions (must respect these)
3. Research implementation patterns, library options, and potential pitfalls
4. Produce RESEARCH.md with:
   - Recommended stack/library choices for this phase
   - Implementation patterns that fit the constraints
   - Known pitfalls and how to avoid them

## Principles

- Respect CONTEXT.md constraints — don't recommend alternatives to locked decisions
- Focus on the specific phase, not the whole project
- Provide concrete, implementation-ready recommendations
